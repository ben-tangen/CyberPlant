# Development Plan

## Core Systems (Build First)
These must exist before anything else works:

### Player
- Movement (walk, jump)
- Collision
- Basic attack
- Dodge / iframe system (if implemented)

### Combat
- Hit detection
- Damage system
- Knockback / feedback

### Enemy System
- Base enemy class
- Movement pattern
- Attack pattern

### Level System
- Scene structure
- Tilemap / collisions
- Basic level flow

---

## Secondary Systems

### Weapon System
- Switch weapons
- Different attack behaviors
- Balance damage / speed

### Resource System
- Water currency
- Pickup logic
- UI updates

### UI
- Health bar
- Currency display
- Main menu

### Boss System
- Unique behaviors
- Multiple phases (if time)
- Pattern-based attacks

### Save System
- Track progress (basic)

---

## Build Order (IMPORTANT)
1. Player movement
2. Combat basics
3. One enemy working
4. One test level
5. UI basics
6. Add more enemies
7. Boss prototype
8. Expand to 3 levels
9. Polish + balance
